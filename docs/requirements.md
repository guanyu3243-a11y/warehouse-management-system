# 功能需求文档

## 文档说明

本文档用于描述服装仓库管理系统的功能需求，作为后续后端、前端、数据库和测试工作的业务依据。

本系统面向服装仓库日常管理场景，重点覆盖用户认证、基础资料维护、入库、出库、库存查询、低库存预警、Dashboard 数据统计和操作日志。

## 用户角色

### 未登录用户

- 可以访问登录页面和注册页面。
- 可以提交注册申请。
- 不可以访问任何仓库业务数据。

### 系统管理员

- 负责系统用户、基础资料和权限范围内的全部业务数据管理。
- 可以管理商品分类、服装商品、仓库、供应商、入库单、出库单、库存数据、低库存预警、Dashboard 和操作日志。
- 可以查看所有用户的操作日志。
- 可以启用、禁用基础资料和用户账号。

### 仓库主管

- 负责仓库业务流程的审核和管理。
- 可以管理商品分类、服装商品、仓库、供应商、入库单、出库单和库存查询。
- 可以确认入库单和出库单。
- 可以查看低库存预警、Dashboard 数据和本仓库相关操作日志。
- 不可以管理系统级用户权限。

### 仓库操作员

- 负责日常入库、出库和库存查询。
- 可以创建和编辑草稿状态的入库单、出库单。
- 可以查询商品、仓库、供应商和库存信息。
- 是否可以确认入库和出库由系统配置决定，默认需要仓库主管或系统管理员确认。
- 不可以删除核心基础资料，不可以查看全部系统日志。

### 只读查看人员

- 只允许查看被授权范围内的数据。
- 可以查看 Dashboard、库存、商品、仓库和供应商信息。
- 不可以新增、修改、删除或确认业务单据。

## 通用业务规则

- 所有业务模块除注册和登录外，都必须在登录后访问。
- 用户登录成功后，系统应识别当前用户身份、角色和权限。
- 新增、修改、删除、确认、取消等关键操作必须写入操作日志。
- 基础资料应支持启用和禁用状态；已被业务单据引用的数据不应被物理删除。
- 系统应对必填字段、唯一字段、数值范围和状态流转进行校验。
- 库存数量不允许小于零。
- 入库和出库必须通过业务单据驱动库存变化，不应直接随意修改库存数量。
- 金额、数量和统计数据应保持一致，单据明细变更后应同步更新单据汇总信息。
- 分页列表应支持按关键字段查询，避免一次性加载过多数据。

## 模块功能需求

### 用户注册和登录

#### 核心功能

- 用户可以通过用户名、密码、邮箱和手机号进行注册。
- 用户可以通过用户名和密码登录系统。
- 登录成功后，系统返回登录状态，并进入后台管理界面。
- 用户可以退出登录。
- 系统可以获取当前登录用户的基础信息，包括用户名、角色和账号状态。
- 管理员可以启用或禁用用户账号。

#### 业务规则

- 用户名必须唯一。
- 密码必须经过安全处理后保存，系统不保存明文密码。
- 禁用状态用户不允许登录。
- 登录状态失效后，用户需要重新登录。
- 注册用户默认角色应为普通仓库操作员，或进入待管理员分配角色的状态。

#### 异常情况

- 用户名已存在时，注册失败并提示原因。
- 用户名或密码错误时，登录失败。
- 用户账号被禁用时，登录失败。
- 未登录访问受保护功能时，系统应提示需要登录。
- 登录状态过期时，系统应提示重新登录。

#### 权限说明

- 未登录用户只能注册和登录。
- 普通用户可以查看自己的登录信息。
- 只有系统管理员可以管理用户状态和角色。

### 商品分类管理

#### 核心功能

- 支持新增商品分类。
- 支持编辑商品分类名称、编码、描述、排序和状态。
- 支持分页查询商品分类列表。
- 支持查看商品分类详情。
- 支持禁用或删除未被引用的商品分类。

#### 业务规则

- 分类编码必须唯一。
- 分类名称不能为空。
- 已被服装商品引用的分类不允许物理删除。
- 禁用分类后，不应再用于新增服装商品。
- 分类可按排序值和创建时间展示。

#### 异常情况

- 分类编码重复时，不允许保存。
- 删除已被商品引用的分类时，应提示该分类正在使用。
- 禁用分类后，如果仍有关联商品，系统应提示影响范围。
- 查询不存在的分类时，应提示数据不存在。

#### 权限说明

- 系统管理员和仓库主管可以新增、编辑、禁用和删除分类。
- 仓库操作员和只读查看人员只能查看分类。

### 服装商品管理

#### 核心功能

- 支持新增服装商品资料。
- 支持维护 SKU、商品名称、分类、尺码、颜色、品牌、季节、成本价、销售价、低库存阈值和状态。
- 支持按 SKU、商品名称、分类、品牌、季节和状态查询商品。
- 支持查看商品详情。
- 支持编辑商品基础信息。
- 支持禁用不再使用的商品。

#### 业务规则

- SKU 必须唯一。
- 商品必须关联有效的商品分类。
- 尺码和颜色是服装商品区分库存的重要属性，应准确维护。
- 成本价、销售价和低库存阈值不能为负数。
- 已产生入库、出库或库存记录的商品不允许物理删除。
- 禁用商品后，不应再创建新的入库或出库明细。
- 低库存阈值用于低库存预警判断。

#### 异常情况

- SKU 重复时，不允许保存。
- 关联的分类不存在或已禁用时，不允许新增商品。
- 成本价、销售价或低库存阈值格式错误时，不允许保存。
- 删除已被业务单据引用的商品时，应提示不能删除。
- 查询不存在的商品时，应提示数据不存在。

#### 权限说明

- 系统管理员和仓库主管可以新增、编辑、禁用和删除商品。
- 仓库操作员可以查看商品，并在创建单据时选择有效商品。
- 只读查看人员只能查看商品。

### 仓库管理

#### 核心功能

- 支持新增仓库资料。
- 支持维护仓库编码、仓库名称、地址、联系人、联系电话和状态。
- 支持分页查询仓库列表。
- 支持查看仓库详情。
- 支持编辑、禁用或删除未被引用的仓库。

#### 业务规则

- 仓库编码必须唯一。
- 仓库名称不能为空。
- 已存在库存、入库单或出库单记录的仓库不允许物理删除。
- 禁用仓库后，不允许创建新的入库单或出库单。
- 库存查询应支持按仓库筛选。

#### 异常情况

- 仓库编码重复时，不允许保存。
- 仓库已被库存或单据引用时，不允许删除。
- 禁用仓库后继续创建单据时，系统应拒绝并提示原因。
- 查询不存在的仓库时，应提示数据不存在。

#### 权限说明

- 系统管理员和仓库主管可以新增、编辑、禁用和删除仓库。
- 仓库操作员和只读查看人员只能查看仓库。

### 供应商管理

#### 核心功能

- 支持新增供应商资料。
- 支持维护供应商编码、名称、联系人、电话、邮箱、地址和状态。
- 支持分页查询供应商列表。
- 支持查看供应商详情。
- 支持编辑、禁用或删除未被引用的供应商。

#### 业务规则

- 供应商编码必须唯一。
- 供应商名称不能为空。
- 已被入库单引用的供应商不允许物理删除。
- 禁用供应商后，不允许用于新建入库单。
- 供应商信息应服务于采购入库追踪。

#### 异常情况

- 供应商编码重复时，不允许保存。
- 删除已被入库单引用的供应商时，应提示不能删除。
- 禁用供应商后继续选择该供应商入库时，系统应拒绝。
- 查询不存在的供应商时，应提示数据不存在。

#### 权限说明

- 系统管理员和仓库主管可以新增、编辑、禁用和删除供应商。
- 仓库操作员可以查看供应商，并在创建入库单时选择有效供应商。
- 只读查看人员只能查看供应商。

### 入库管理

#### 核心功能

- 支持创建入库单。
- 入库单应包含入库仓库、供应商、操作人、备注和商品明细。
- 入库明细应包含商品、数量、单件成本和备注。
- 支持编辑草稿状态入库单。
- 支持查询入库单列表和入库单详情。
- 支持确认入库。
- 支持取消草稿或未确认状态的入库单。
- 确认入库后，系统自动增加对应仓库中对应商品的库存。

#### 业务规则

- 入库单应有唯一单号。
- 入库数量必须大于零。
- 入库商品必须是有效商品。
- 入库仓库必须是有效仓库。
- 入库供应商必须是有效供应商。
- 草稿状态的入库单可以修改。
- 已确认或已取消的入库单不允许修改明细。
- 入库单只能确认一次，重复确认不应重复增加库存。
- 确认入库应作为完整事务处理，单据状态和库存变化必须同时成功或同时失败。

#### 异常情况

- 入库仓库不存在或已禁用时，不允许创建或确认入库单。
- 供应商不存在或已禁用时，不允许创建或确认入库单。
- 商品不存在或已禁用时，不允许加入入库明细。
- 入库数量为空、为零或为负数时，不允许保存。
- 重复确认入库单时，系统应拒绝。
- 入库确认过程中库存更新失败时，应回滚本次确认操作。

#### 权限说明

- 系统管理员和仓库主管可以创建、编辑、确认和取消入库单。
- 仓库操作员可以创建和编辑草稿入库单；是否可以确认入库由系统配置决定。
- 只读查看人员只能查看入库单。

### 出库管理

#### 核心功能

- 支持创建出库单。
- 出库单应包含出库仓库、操作人、备注和商品明细。
- 出库明细应包含商品、数量、单件销售价和备注。
- 支持编辑草稿状态出库单。
- 支持查询出库单列表和出库单详情。
- 支持确认出库。
- 支持取消草稿或未确认状态的出库单。
- 确认出库后，系统自动扣减对应仓库中对应商品的库存。

#### 业务规则

- 出库单应有唯一单号。
- 出库数量必须大于零。
- 出库商品必须是有效商品。
- 出库仓库必须是有效仓库。
- 确认出库前必须校验库存是否充足。
- 库存不足时，不允许确认出库。
- 已确认或已取消的出库单不允许修改明细。
- 出库单只能确认一次，重复确认不应重复扣减库存。
- 确认出库应作为完整事务处理，单据状态和库存变化必须同时成功或同时失败。

#### 异常情况

- 出库仓库不存在或已禁用时，不允许创建或确认出库单。
- 商品不存在或已禁用时，不允许加入出库明细。
- 出库数量为空、为零或为负数时，不允许保存。
- 当前库存小于出库数量时，不允许确认出库。
- 重复确认出库单时，系统应拒绝。
- 出库确认过程中库存更新失败时，应回滚本次确认操作。

#### 权限说明

- 系统管理员和仓库主管可以创建、编辑、确认和取消出库单。
- 仓库操作员可以创建和编辑草稿出库单；是否可以确认出库由系统配置决定。
- 只读查看人员只能查看出库单。

### 库存查询

#### 核心功能

- 支持按仓库、商品名称、SKU、分类和是否低库存查询库存。
- 支持查看某个商品在各仓库的库存分布。
- 支持查看某个仓库下所有商品库存。
- 库存列表应展示商品、SKU、分类、仓库、当前库存、锁定库存、可用库存和低库存状态。
- 支持分页查询库存列表。

#### 业务规则

- 库存由入库和出库业务自动产生和更新。
- 同一商品在同一仓库只能有一条库存记录。
- 可用库存等于当前库存减去锁定库存。
- 当前库存和锁定库存不能为负数。
- 库存查询结果应与已确认入库单和已确认出库单保持一致。

#### 异常情况

- 查询条件对应的数据不存在时，应返回空列表而不是系统错误。
- 商品或仓库不存在时，应提示查询对象不存在。
- 库存数据出现负数时，应视为异常数据，需要管理员排查。

#### 权限说明

- 系统管理员、仓库主管、仓库操作员和只读查看人员均可查看授权范围内的库存。
- 普通仓库操作员只能查看其被授权仓库的库存，具体范围由系统配置决定。

### 低库存预警

#### 核心功能

- 系统根据商品低库存阈值判断低库存状态。
- 支持查询低库存商品列表。
- 支持在 Dashboard 中展示低库存商品数量和重点预警商品。
- 支持按仓库、分类、SKU 和商品名称筛选低库存商品。
- 低库存列表应展示商品、仓库、当前库存、可用库存和预警阈值。

#### 业务规则

- 当商品在某仓库的可用库存小于或等于低库存阈值时，应进入低库存预警。
- 低库存阈值为零时，表示只有库存为零或低于零的异常情况才触发预警。
- 禁用商品可以不进入常规预警列表，但应保留历史库存查询能力。
- 入库确认后，如果库存恢复到阈值以上，应自动解除低库存预警状态。
- 出库确认后，如果库存降到阈值以内，应自动出现在低库存预警列表中。

#### 异常情况

- 商品未设置低库存阈值时，系统应按默认阈值处理或提示资料不完整。
- 库存数据异常为负数时，应优先展示为异常预警。
- 仓库或商品被禁用时，应在预警列表中明确状态，避免误操作。

#### 权限说明

- 系统管理员、仓库主管、仓库操作员和只读查看人员均可查看授权范围内的低库存预警。
- 只有系统管理员和仓库主管可以调整商品低库存阈值。

### Dashboard 数据统计

#### 核心功能

- 展示商品总数、分类总数、仓库总数、供应商总数。
- 展示当前库存总量。
- 展示低库存商品数量。
- 展示今日入库数量和今日出库数量。
- 展示近期入库和出库趋势。
- 展示低库存商品排行。
- 展示近期关键操作日志。

#### 业务规则

- Dashboard 数据应基于当前用户权限范围统计。
- 入库数量只统计已确认入库单。
- 出库数量只统计已确认出库单。
- 今日统计应按系统统一日期口径计算。
- 低库存统计应与低库存预警模块保持一致。
- 统计数据应尽量保证实时性，但可以根据后续性能需要增加缓存策略。

#### 异常情况

- 没有业务数据时，应展示零值或空状态。
- 当前用户无权限查看某类数据时，不应展示对应明细。
- 统计计算失败时，应提示数据加载失败，不影响其他模块访问。

#### 权限说明

- 系统管理员可以查看全系统统计数据。
- 仓库主管可以查看其管理范围内的统计数据。
- 仓库操作员可以查看其授权仓库范围内的基础统计数据。
- 只读查看人员只能查看被授权范围内的统计数据。

### 操作日志

#### 核心功能

- 系统记录用户登录、退出、创建、修改、删除、确认、取消等关键操作。
- 日志应包含操作用户、模块、操作类型、请求路径、操作描述、请求 IP 和操作时间。
- 支持按用户、模块、操作类型和时间范围查询日志。
- 支持查看日志详情。
- 支持分页查询日志列表。

#### 业务规则

- 操作日志应由系统自动记录，普通用户不应手动新增或修改日志。
- 关键业务操作无论成功或失败，都应尽量记录操作结果。
- 日志记录不应影响主业务流程；日志写入失败时，应记录系统异常，不能导致正常业务不可用。
- 操作日志原则上不允许删除，后续如需清理应通过管理员配置归档策略。

#### 异常情况

- 日志查询条件无匹配结果时，应返回空列表。
- 查询不存在的日志详情时，应提示数据不存在。
- 日志写入失败时，应保留业务结果，并由系统记录异常信息供排查。

#### 权限说明

- 系统管理员可以查看全部操作日志。
- 仓库主管可以查看其管理范围内的业务操作日志。
- 仓库操作员只能查看自己的操作日志或被授权范围内的日志。
- 只读查看人员默认不允许查看操作日志，除非被单独授权。

## 权限矩阵

## 非功能性需求

- 系统界面应清晰展示服装商品的尺码、颜色、品牌和季节等关键属性。
- 列表查询应支持分页、筛选和关键字搜索。
- 表单提交应有明确的成功、失败和校验提示。
- 关键操作如删除、禁用、确认入库、确认出库和取消单据，应进行二次确认。
- 系统应保证库存变更的一致性，避免并发操作导致库存错误。
- 系统应具备基础审计能力，便于追踪用户操作和排查问题。
