# 服装仓库管理系统用户指南

本文档面向商品管理员和仓库操作人员，重点说明如何使用公司现有库存统计表或校服库存数量表，一次性创建服装商品规格并初始化库存。系统当前按单仓库模式运行。

## 一、功能用途

公司数量统计表使用“型号 + 颜色 + 多个尺码数量”的矩阵结构。系统会把每个型号和颜色自动展开为十个独立商品：

```text
XS、S、M、L、XL、2XL、3XL、4XL、5XL、6XL
```

例如表格中的型号 `V3`、颜色 `白色` 会创建：

```text
V3-白色-XS
V3-白色-S
V3-白色-M
...
V3-白色-6XL
```

即使某个尺码数量为空或为 `0`，系统也会创建该商品规格和零库存记录。

校服库存数量表使用“名称 + 尺码列”的矩阵结构。系统会把表格标题作为颜色/款式字段，例如 `校服赣`，并按列头创建尺码：

```text
短袖T恤-校服赣-115#
短袖T恤-校服赣-120#
短袖T恤-校服赣-125#
...
```

## 二、导入前准备

导入前请确认：

1. 系统中只有一个状态为 `ACTIVE` 的仓库。
2. 已在“商品分类”中创建至少一个状态为 `ACTIVE` 的分类。
3. Excel 每行的尺码数量正确；公司数量统计表还需要确认“合计”列正确。
4. 正式环境已完成数据库备份，或先在测试环境试导入。

不需要提前逐条创建商品。导入弹窗中选择的分类会应用于本次新创建的全部商品。

## 三、支持的文件格式

系统支持：

- Excel 97-2003 工作簿：`.xls`
- Excel 工作簿：`.xlsx`

公司现用的 `2026年6月5日更新数量统计表(1).xls` 可以直接上传。系统读取第一组以下结构：

```text
型号 | 颜色 | XS | S | M | L | XL | 2XL | 3XL | 4XL | 5XL | 6XL | 合计
```

解析规则：

- 第一行报表标题可以保留，系统会自动查找表头。
- 型号为空时继承上一行非空型号。
- 尺码数量为空时按 `0` 处理。
- 数量必须是非负整数。
- 每行“合计”必须等于十个尺码数量之和。
- `合计`、`合计总量` 和空白行不会创建商品。

校服库存数量表 `校服赣-库存数量.xlsx` 也可以直接上传。系统读取以下结构：

```text
名称 | 115# | 120# | 125# | 130# | 135# | 140# | ... | 180#
```

解析规则：

- 表格上方的标题会作为颜色/款式字段，例如 `校服赣`。
- 第一列 `名称` 会作为商品名称。
- 数字尺码列会作为商品尺码，例如 `115#`、`120#`。
- 尺码数量为空时按 `0` 处理。
- 数量必须是非负整数。

## 四、导入操作

1. 登录系统。
2. 进入“基础资料 > 服装商品”。
3. 点击页面右上角“导入公司商品表”。
4. 选择本次新商品所属的启用分类。
5. 选择公司数量统计表或校服库存数量表 `.xls` / `.xlsx` 文件。
6. 阅读商品创建、库存覆盖和事务回滚说明。
7. 点击“开始导入”。
8. 在确认弹窗中点击“确认导入”。
9. 等待系统完成整份文件校验、商品创建和库存初始化。

成功后弹窗显示：

- 导入批次号
- 商品分类
- 目标仓库
- 规格总数
- 新建商品数
- 复用商品数
- 新建库存记录数
- 更新库存数
- 库存未变化数
- 零库存规格数

服装商品列表会自动刷新。

## 五、商品创建规则

每个商品规格按以下方式生成：

| 商品字段 | 导入值 |
| --- | --- |
| SKU | `商品名称-颜色或款式-尺码` |
| 商品名称 | 公司表的 Excel 型号，或校服表的 Excel 名称 |
| 分类 | 导入弹窗中选择的分类 |
| 颜色 | 公司表的 Excel 颜色，或校服表上方标题 |
| 尺码 | 当前尺码列 |
| 品牌 | 空 |
| 季节 | 空 |
| 成本价 | `0` |
| 销售价 | `0` |
| 预警阈值 | `0` |
| 状态 | `ACTIVE` |

重复导入时：

- 相同 SKU 已存在且商品名称、颜色、尺码一致：复用已有商品。
- 复用商品的分类、品牌、季节、价格和预警阈值保持原值。
- 相同 SKU 对应的名称、颜色或尺码不一致：整次导入失败。
- 相同 SKU 商品已禁用：整次导入失败，系统不会自动启用。

## 六、库存与流水规则

- Excel 数量是导入后的库存总量，不是增加数量。
- 系统自动写入唯一启用仓库。
- 新商品会创建库存记录，包括数量为 `0` 的规格。
- 已有商品库存会覆盖为 Excel 中的目标数量。
- 系统不会物理删除商品或库存。
- 库存发生变化时会生成库存流水。
- 流水类型为 `IMPORT`，来源类型为 `COMPANY_PRODUCT_IMPORT`。
- 同一次导入的流水使用相同批次号。
- 数量未变化时不生成重复流水。

导入后可进入“库存管理 > 库存流水”，按批次号核对本次变化。

## 七、整表事务与失败处理

导入采用整表原子事务。系统会先校验完整文件和已有数据，全部通过后才写入商品、库存和流水。

以下任一情况都会使整次导入失败并回滚：

- 所选分类不存在或已禁用。
- 系统中不是恰好一个启用仓库。
- Excel 表头不符合支持的公司数量统计表或校服库存数量表模板。
- Excel 中存在重复的商品名称、颜色或款式、尺码规格。
- 自动生成的 SKU 超过 80 个字符。
- 已有相同 SKU 商品资料与 Excel 冲突。
- 已有相同 SKU 商品已禁用。
- 数量不是非负整数。
- 公司数量统计表的行合计与十个尺码数量之和不一致。
- 目标库存小于该商品当前锁定库存。
- 商品、库存或库存流水写入失败。

失败时不会留下部分商品或部分库存。根据页面提示修正分类、已有商品或 Excel 后重新导入。

## 八、常见问题

### 1. 为什么导入后商品数量很多

系统会为每个“型号 + 颜色”创建全部十个尺码，包括零库存规格。这是为了以后补货时无需再次创建商品。

### 2. 提示 Existing product conflicts with Excel

系统中已存在相同 SKU，但商品名称、颜色或尺码与 Excel 不一致。请先检查并修正已有商品，避免库存写入错误规格。

### 3. 提示 Existing product is disabled

相同 SKU 商品已被禁用。请确认该商品是否应该恢复启用，再通过商品编辑功能处理。

### 4. 提示行合计不一致

检查提示行中 `XS` 到 `6XL` 的数量之和，并修正“合计”列。

### 5. 提示数量低于锁定库存

该规格存在已锁定库存，导入后的库存总量不能小于锁定数量。请先核对出库流程或调整 Excel 数量。

### 6. 导入后某些规格库存为 0

模板中的空白数量按 `0` 处理，且系统会创建全部尺码。这属于已确认的导入规则。

## 九、正式环境操作建议

- 首次导入先使用测试环境和完整数据库备份。
- 导入前确认没有人员正在集中处理出库、盘点或库存调整。
- 导入后抽查多个型号、颜色和尺码，并核对商品列表、库存查询和库存流水。
- 保存原始 Excel 文件和系统返回的导入批次号。
- 发现库存问题时不要直接修改数据库，应使用库存调整或修正 Excel 后重新导入。
